# Group Collaboration Workflow
**Internal Guidelines — The Avengers (Group H)**

This document defines how we work together to ensure our code integrates without conflicts.

## 1. Git & Branching Strategy
- `main` is protected.
- Each student works in their feature branch.
- PR review before merge.

### Branches
- `feat/riagul-teams`
- `feat/lucas-org`
- `feat/suliman-msgs`
- `feat/maurya-schedule`
- `feat/hussain-reports`

## 2. Peer Review Rules
- Extend `base.html`.
- Reuse shared style tokens.
- Remove debug prints.

## 3. Daily Integration
- Lead reviews and merges approved PRs.
- Everyone syncs latest main/develop branch and migrations.

## 4. Folder Ownership
- Riagul: `teams/`
- Lucas: `organisation/`
- Suliman: `messages_app/`
- Maurya: `schedule/` and shared integration
- Hussain: `reports/`
